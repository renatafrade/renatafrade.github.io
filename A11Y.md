# Accessibility (a11y) testing

## Automated test (axe-core)

Once dependencies are installed, run:

```bash
npm install   # if not already done (axe-core, jsdom)
npm run test:a11y
```

This runs [axe-core](https://github.com/dequelabs/axe-core) against all HTML pages (WCAG 2.0/2.1 Level A & AA, best-practice). Exit code 1 if any violations are found.

---

## Manual checklist

Use this with a screen reader and keyboard-only navigation.

### Structure & semantics

- [ ] **One `<h1>` per page** – only the main page title
- [ ] **Heading order** – no skipped levels (h1 → h2 → h3)
- [ ] **Landmarks** – `<main id="main-content">`, `<nav aria-label="Main navigation">`, footer present
- [ ] **Skip link** – “Skip to main content” appears on first Tab; moves focus to main content
- [ ] **Page title** – unique, descriptive `<title>` on every page
- [ ] **Lang** – `<html lang="en">` set

### Images & media

- [ ] **Alt text** – every meaningful image has descriptive `alt`; decorative images use `alt=""` or `role="presentation"`
- [ ] **Profile image** – alt describes person/context (e.g. “Renata Loureiro Frade, PhD — researcher…”)
- [ ] **Dimensions** – important images have `width` and `height` to reduce layout shift

### Links & buttons

- [ ] **Link purpose** – link text (or `aria-label`) makes sense without surrounding context; avoid “click here”
- [ ] **External/PDF** – `target="_blank"` links have `rel="noopener noreferrer"`; “opens in new tab” in label/aria-label where helpful (e.g. CV)
- [ ] **Buttons** – menu toggle has `aria-expanded`, `aria-controls="nav-dropdown"`, `aria-label="Toggle menu"`

### Forms & focus

- [ ] **Focus visible** – all interactive elements show a clear focus ring (e.g. `outline` on `:focus-visible`); check with Tab only
- [ ] **No focus trap** – Tab cycles through nav, main content, footer; Escape closes mobile menu
- [ ] **Focus order** – matches visual order (skip link → nav → main → footer)

### Keyboard

- [ ] **Tab** – reach skip link, all nav links, CV, and main content links
- [ ] **Enter/Space** – activate links and menu button
- [ ] **Escape** – closes mobile nav dropdown
- [ ] **No keyboard traps** – focus never stuck

### Color & contrast

- [ ] **Contrast** – text meets WCAG AA (4.5:1 normal text, 3:1 large); purple accent on white/light background checked
- [ ] **Not color-only** – links/buttons identifiable by more than color (underline, focus ring, etc.)

### Responsive & zoom

- [ ] **Zoom** – at 200% zoom, no horizontal scroll; content still usable
- [ ] **Touch targets** – mobile nav items and key buttons at least ~44×44px
- [ ] **Viewport** – `meta name="viewport" content="width=device-width, initial-scale=1.0"` present

### Screen reader (optional)

- [ ] **NVDA / VoiceOver / JAWS** – announce headings, landmarks, link names, and button state correctly
- [ ] **Mobile menu** – expanded/collapsed state announced; list of links readable

## Quick browser checks

1. **Chrome DevTools** – Lighthouse → Accessibility category (needs Chrome).
2. **Firefox** – Accessibility panel in DevTools; check contrast and keyboard.
3. **axe DevTools** – [Browser extension](https://www.deque.com/axe/devtools/) (Chrome, Edge, Firefox) to run axe on the live page.
